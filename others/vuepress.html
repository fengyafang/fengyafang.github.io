      # 创建博客

      1. 建立一个文件夹 然后进入里面执行
      npm init

      进行初始化 得到package.json文件

      1. 安装vuepress依赖
      npm install -D vuepress
      1. 创建docs文件夹 在这个文件夹里面建一个README.md文档
      mkdir docs && echo '# Hello VuePress' > docs/README.md
      1. 在package.json中的scripts对象中添加两个属性dev,build
      {
  "scripts": {
    "dev": "vuepress dev docs",
    "build": "vuepress build docs"
  }
}
      1. 启动服务
      npm run dev

      TIP

      启动前doc以及里面的README.md文件一定要存在 不然会报错

      # 部署到githubpage

      1. 新建一个仓库 如果仓库名是username.github.io username是你的github用户名. 访问链接为username.github.io 如果随便起一个仓库名 访问链接为 username.github.io/你的仓库名 这种连接需要配置base配置方法如下

        1. 在docs目录下创建.vuepress文件夹
        2. 进入文件夹并创建config.js文件 内容如下
        module.exports = {
	base:'/你的仓库名/'
}

      2. 在你的项目中，创建一个如下的 deploy.sh 文件




















       


       



      #!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run build

# 进入生成的文件夹
cd docs/.vuepress/dist

# 如果是发布到自定义域名
# echo 'www.example.com' > CNAME

git init
git add -A
git commit -m 'deploy'

# 如果发布到 https://<USERNAME>.github.io
# git push -f git@github.com:<USERNAME>/<USERNAME>.github.io.git master

# 如果发布到 https://<USERNAME>.github.io/<REPO>
# git push -f git@github.com:<USERNAME>/<REPO>.git master:gh-pages

cd -

      如果是第一种仓库 把第一个高亮行中的仓库地址换成你的仓库地址并取消注释 第二种仓库就把 第二个高亮行的仓库地址换成你的,然后把第二行取消注释

      1. 在package.json中的scripts对象添加一个属性deploy
      "scripts": {
    "dev": "vuepress dev docs --temp .temp",
    "build": "vuepress build docs",
	"deploy":"bash deploy.sh"
  }
      1. 执行以下命令 然后就可以用第一步中的链接来访问你的博客了
      npm run deploy
